# Contributing to FlowDesk

Thank you for your interest in contributing to FlowDesk! 

Because FlowDesk is a proprietary, closed-source project whose source code is made available publicly strictly for educational and portfolio purposes, we have very strict guidelines regarding contributions.

1. **Assignment of Copyright**: You irrevocably assign, transfer, and convey to the project owner (Kharis Destian Maulana / Riray) all worldwide rights, titles, and interests in and to your contributions, including all copyrights and other intellectual property rights.
2. **Waiver of Rights**: You waive any right to claim ownership over the contributed code or to demand its removal from the FlowDesk repository or application in the future.
3. **No Compensation**: You understand that contributions are voluntary, and you are not entitled to any financial compensation, royalties, or equity for your submissions.
4. **Original Work**: You guarantee that your contribution is your own original work and that you have the right to assign the copyright to the project owner. You must not submit code that belongs to your employer or any third party unless you have explicit written permission to assign it to this project.

## How to Contribute

1. **Request Permission First**: Since this is a proprietary project, unsolicited large PRs may be rejected. If you wish to contribute, please open an Issue first to discuss the feature or bug fix with the owner.
2. **Fork the Repository**: Create a fork of the repository for your changes.
3. **Submit a Pull Request**: When your code is ready, submit a Pull Request. By doing so, you automatically agree to the CLA above.

If you have any questions or are unsure about these terms, please contact the repository owner before submitting any code.
